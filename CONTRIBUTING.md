# Contributing to VedWriter

Thanks for helping make private journaling better.

## Local setup

```bash
npm install
npm run dev
```

Before submitting a change, run:

```bash
npm run lint
npm run build
```

Good first contributions include documentation, translations, accessibility improvements, encryption/backup tests, import/export support, and reproducible UI fixes.

Keep pull requests focused. Explain the user problem, the change, and how it was tested. Never include real journal content, passwords, exported backups, or other private data in issues, commits, or screenshots. Changes involving cryptography, password handling, backup formats, or data deletion should include tests and a security-focused explanation.
